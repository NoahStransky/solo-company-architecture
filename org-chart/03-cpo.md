# 📱 CPO / PM — 产品负责人

> "用户真正需要什么？"

## 🎯 核心职责

一人公司的产品经理不是画原型的美工，而是**用户的代言人**——确保你写的每一行代码都在解决真实问题。

### 1. 用户研究（User Research）
- 定义目标用户画像（Persona）
- 收集用户反馈（访谈、问卷、行为数据）
- 分析竞品（Competitive Analysis）

### 2. 需求管理（Requirement Management）
- 收集需求 → 分类（Bug / Feature / Optimization）
- 优先级排序（RICE / MoSCoW 方法）
- 写用户故事（User Stories）

### 3. 产品设计（Product Design）
- 信息架构（Information Architecture）
- 用户流程（User Flow）
- 低保真原型（Lo-fi Wireframes）

### 4. 数据驱动（Data-Driven）
- 定义核心指标（North Star Metric）
- A/B 测试设计
- 漏斗分析（Acquisition → Activation → Retention → Revenue → Referral）

---

## 🎛️ 决策权限

| 决策类型 | CPO 权限 | 备注 |
|---------|---------|------|
| 功能优先级 | ✅ 最终决定 | 基于用户价值和开发成本 |
| 产品设计 | ✅ 最终决定 | UI 细节与设计师协商 |
| 用户体验 | ✅ 完全授权 | 技术限制内优化体验 |
| 发布内容 | ⚠️ 协商决定 | 与 Growth 共同决定 |
| 技术可行性 | ❌ 不决定 | 需 CTO 确认 |
| 定价策略 | ❌ 不决定 | CEO 决策，CPO 提供数据支持 |

## 📐 关键产出物

### 1. 产品需求文档（PRD）模板

```markdown
# PRD-007: AI 新闻摘要功能

## 背景
用户反馈每天看大量科技新闻耗时太长，希望有自动摘要。

## 目标
- 减少用户阅读时间 50%
- 提高用户日活跃度 20%

## 用户故事
作为 [IT 从业者]，
我希望 [每天收到精选的 AI 新闻摘要]，
以便 [5 分钟了解行业动态]。

## 功能范围
### In Scope
- [ ] 从 5 个数据源抓取新闻
- [ ] AI 生成 1-2 句话摘要
- [ ] 按类别分组展示

### Out of Scope
- [ ] 用户自定义数据源（V2）
- [ ] 邮件订阅（V2）

## 设计
[附上原型图或文字描述]

## 指标
- 摘要点击率 > 60%
- 平均阅读时长 > 30s

## 技术备注
- 使用 OpenRouter API
- 预计延迟 2-3 秒
- 预估成本 $0.01/用户/天
```

### 2. 用户画像（Persona）

```markdown
## Persona: "忙碌的技术负责人 Alex"

- **年龄**: 32 岁
- **职位**: 初创公司 CTO
- **技术栈**: Python, React, AWS
- **痛点**:
  1. 没时间跟踪所有 AI 新闻
  2. 信息过载，不知道哪些重要
  3. 希望快速了解 "对我工作有什么影响"
- **使用场景**: 每天早上通勤时用手机浏览
- **目标**: 10 分钟内了解今天最重要的 3-5 件事
```

### 3. 竞品分析矩阵

| 竞品 | 优势 | 劣势 | 我们的机会 |
|------|------|------|-----------|
| TechCrunch | 品牌大，速度快 | 广告多，不够个性化 | 更聚焦 AI/程序员 |

## ⚠️ PM 常见陷阱

1. **"我觉得用户需要"**
   - 没有数据支撑的功能不做
   - 先做用户访谈，再写 PRD

2. **追求功能数量**
   - 一个产品的成功 = 核心功能做到极致
   - 功能越多，维护成本指数级增长

3. **忽视技术约束**
   - 设计前和 CTO 确认可行性
   - 了解 API 延迟、成本限制
