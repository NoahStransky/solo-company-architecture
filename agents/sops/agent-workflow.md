# 🔄 Agent 协作 SOP

> 标准操作流程：如何让多个 AI Agent 高效协作完成一个项目

---

## 📋 流程总览

```
CEO 提出需求
    ↓
Secretary 接收 & 拆解
    ↓
Phase 1: 需求澄清（CPO Agent）
    ↓
Phase 2: 技术评估（CTO Agent）
    ↓
Phase 3: 执行开发（Dev Agent）
    ↓
Phase 4: 质量审查（CTO Agent）
    ↓
Phase 5: 推广准备（Growth Agent）
    ↓
Secretary 汇总汇报 → CEO
```

---

## Phase 1: 需求澄清（CPO Agent）

**触发条件**：CEO 提出新需求

**执行者**：CPO Agent

**输入**：
- CEO 的原始需求描述
- 项目背景信息
- 目标用户（如有）

**输出**：
- 需求分析报告
- 用户画像
- 功能优先级建议
- PRD 草案

**检查点**：
- [ ] 需求是否清晰可执行？
- [ ] 用户画像是否合理？
- [ ] 功能优先级是否符合商业目标？

**升级条件**：
- 需求与现有产品方向冲突 → 升级 CEO
- 技术不可行（需 CTO 预评估）→ 并行启动 CTO Agent

---

## Phase 2: 技术评估（CTO Agent）

**触发条件**：CPO Agent 完成 PRD

**执行者**：CTO Agent

**输入**：
- CPO Agent 的 PRD
- 现有技术架构文档
- 约束条件（预算、时间、技术栈）

**输出**：
- 技术方案文档
- 工作量估算
- 风险评估
- 技术栈建议

**检查点**：
- [ ] 技术方案是否简洁可行？
- [ ] 工作量估算是否合理？
- [ ] 风险是否有缓解方案？

**升级条件**：
- 技术方案与现有架构冲突 → 升级 CEO
- 工作量超出预期 → 提供替代方案 + 升级 CEO

---

## Phase 3: 执行开发（Dev Agent）

**触发条件**：CEO 确认技术方案

**执行者**：Dev Agent

**输入**：
- 确认的 PRD
- 技术方案文档
- 代码规范
- 现有代码库

**输出**：
- 实现代码
- 单元测试
- 更新文档
- 部署说明

**检查点**：
- [ ] 代码是否遵循规范？
- [ ] 测试是否通过？
- [ ] 文档是否更新？

**升级条件**：
- 实现过程中发现需求不清晰 → 退回 CPO Agent
- 技术方案需要调整 → 退回 CTO Agent
- 遇到阻塞超过 2 小时 → 升级 Secretary

---

## Phase 4: 质量审查（CTO Agent）

**触发条件**：Dev Agent 完成开发

**执行者**：CTO Agent

**输入**：
- Dev Agent 的代码输出
- PRD 验收标准
- 安全/性能要求

**输出**：
- 代码审查报告
- 修改建议（如有）
- 是否通过的结论

**检查点**：
- [ ] 是否通过代码审查？
- [ ] 安全漏洞是否修复？
- [ ] 性能是否符合要求？

**升级条件**：
- 审查不通过 → 退回 Dev Agent 修改（最多 2 轮）
- 发现重大架构问题 → 升级 CEO

---

## Phase 5: 推广准备（Growth Agent）

**触发条件**：代码审查通过

**执行者**：Growth Agent

**输入**：
- 新功能说明
- 目标用户
- 竞品对比（Analyst Agent 提供）

**输出**：
- 推广文案
- 内容日历更新
- 发布计划

**检查点**：
- [ ] 文案是否准确描述功能？
- [ ] 是否有明确的 CTA？
- [ ] 发布渠道是否覆盖目标用户？

---

## 🔄 并行执行规则

某些任务可以并行执行，缩短总时间：

| 主任务 | 可并行任务 | 节省 |
|--------|-----------|------|
| CPO 写 PRD | CTO 预评估技术可行性 | 0.5 天 |
| Dev 开发 | Growth 准备推广素材 | 1 天 |
| Dev 自测 | Analyst 做竞品更新分析 | 0.5 天 |

---

## ⚠️ 异常处理

### Agent 输出质量不达标
1. Secretary 给出具体修改意见
2. Agent 重新执行（最多 2 次）
3. 仍不达标 → 升级 CEO

### Agent 之间意见冲突
1. Secretary 收集双方论据
2. 给出建议方案
3. 仍无法达成一致 → 升级 CEO 决策

### 需求变更
1. Secretary 评估影响范围
2. 通知相关 Agent 调整
3. 更新工作量估算
4. 告知 CEO 变更影响
