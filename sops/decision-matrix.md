# 🎯 决策矩阵 — Solo Company Decision Framework

> 一人公司最大的挑战：**决策疲劳**。这个框架帮你快速判断"谁来做决定"。

## ⚡ 30 秒决策法则

遇到任何决策，问自己 3 个问题：

1. **这会影响公司生存吗？**（钱、法律、核心用户）
   - Yes → **CEO 决定**
2. **这需要改代码或技术架构吗？**
   - Yes → **CTO 决定**
3. **这会影响用户体验吗？**
   - Yes → **CPO 决定**

如果以上都是 No → **Growth 或执行角色决定**，不需要开会。

| 场景 | 决策者 | 时间限制 | 需要输入 |
|------|--------|---------|---------|

| 用 React 还是 Vue | CTO | 48h | 团队熟悉度、生态 |
| 先做功能 A 还是 B | CPO | 24h | 用户价值评估 |
| 这篇推文怎么写 | Growth | 即时 | 品牌调性指南 |

| 网站颜色从蓝改绿 | CPO | 即时 | 品牌一致性 |

## 🔄 决策流程图

```
开始
  │
  ▼
是否涉及公司生存/法律/重大财务？
  │
  ├─ Yes ──→ CEO 决定（可咨询他人）
  │            ↓
  │         记录决策原因
  │            ↓
  │         执行
  │
  └─ No ───→ 是否涉及技术实现？
               │
               ├─ Yes ──→ CTO 决定
               │            ↓
               │         如果涉及用户体验 → 咨询 CPO
               │            ↓
               │         执行
               │
               └─ No ───→ 是否涉及产品功能/用户体验？
                            │
                            ├─ Yes ──→ CPO 决定
                            │            ↓
                            │         如果涉及技术限制 → 咨询 CTO
                            │            ↓
                            │         执行
                            │
                            └─ No ───→ Growth/执行者决定
                                         ↓
                                      执行
```

## 📝 决策记录模板

**重要决策必须记录**，否则你会忘记当初为什么这样选。

```markdown
## Decision-012: 选择 Stripe 而非 Paddle 作为支付网关

### 日期
2026-04-23

### 决策者
CEO (咨询 CTO 后)

### 背景
需要为 SaaS 产品接入订阅支付。

### 选项
| 选项 | 优势 | 劣势 |
|------|------|------|
| Stripe | 文档好，API 干净，费率低 | 需要处理税务 |
| Paddle | 代扣代缴税务，Merchant of Record | 费率贵 5%，提现慢 |

### 决策
选择 Stripe。

### 原因
1. 费率差异每年可省 ¥5000+
2. CTO 有 Stripe 集成经验，3 天可上线
3. 税务可用第三方服务（TaxJar）补充

### 风险
- 需要自己处理部分税务合规
- 缓解：先专注国内市场，国际扩张时再评估

### 回顾日期
2026-07-23（3个月后复盘）
```

## 💡 提高决策速度的技巧

### 1. 默认值法则（Default Choices）
预先设定好默认值，避免每次重新思考：
- 默认云服务：AWS / Vercel
- 默认数据库：PostgreSQL
- 默认前端框架：React
- 默认不写测试的场景：一次性脚本

### 2. 2-way door 原则（Jeff Bezos）
- **单向门**（不可逆）：慎重决策，多方咨询
- **双向门**（可逆）：快速决策，错了再改
- 一人公司 90% 的决策都是双向门

### 3. 时间盒（Time Boxing）
给每个决策设定时间上限：
- 日常技术决策：5 分钟
- 功能优先级：30 分钟
- 架构选型：2 小时
- 商业模式：1 天

时间到了必须决策，perfect is the enemy of done。
